DELETING SENSITIVE DATA FROM GITHUB COMMITS:

Commits always take a record of what you have done to your repository and if you delete a password, they still can be seen in commits. One way to retouch commits is to use BFG Repo-Cleaner. the following codes should be written in the Bash terminal. First You should have installed "Java", then download "bfg.jar" file."I placed it in /c/DEVELOPMENT."      Docs

1 . Make a  text file of the passwords you want to delete from commits. "p.txt" "each line, one password"


8BYkEfBA6O6donzWlSihBXox7C0sKR6b
2 . Make a clone of the repository.  

FaMiLy@DESKTOP-8OKIJ2K MINGW64 ~/Documents/GitHub
$ git clone https://github.com/Id-Dark-Dragon/website.git

3 . Using a command line to replace texts in p.txt by "--REMOVED--" on commits. 

FaMiLy@DESKTOP-8OKIJ2K MINGW64 /c/DEVELOPMENT
$ java -jar bfg.jar --replace-text "E:\PRJ\p.txt" "C:\Users\FaMiLy\Documents\GitHub\website\.git"

4 . Go to .git and force push it to GitHub. 

FaMiLy@DESKTOP-8OKIJ2K MINGW64 ~/Documents/GitHub/website/.git (GIT_DIR!)
$ git push origin --force --all
